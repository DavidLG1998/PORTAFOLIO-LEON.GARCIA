Semana 9: Desarrollo de Aplicaciones Web Backend con PHP y JSP

Descripción de los temas aprendidos

Durante esta semana se abordó la tecnología Web del lado del servidor (Backend), comprendiendo los fundamentos de cómo funciona una aplicación web desde el servidor. Se desarrollaron los siguientes temas clave: Arquitectura de aplicaciones web: se estudió la estructura de una aplicación web moderna, identificando la interacción entre cliente, servidor, base de datos y navegador. Funcionamiento del server-side: se analizó cómo el servidor procesa las solicitudes, ejecuta lógica de negocio y responde con contenido dinámico al navegador. Lenguajes y frameworks backend: se introdujeron los principales lenguajes usados en el desarrollo del backend, como PHP y Java con JSP (Java Server Pages), incluyendo sus características y casos de uso. Desarrollo práctico: Aplicaciones web con PHP: creación de scripts para generar contenido dinámico, manejo de formularios y conexión con base de datos. Aplicaciones web con JSP: desarrollo de páginas dinámicas en Java, empleando servlets y JSTL para la generación de vistas desde el servidor. El Laboratorio 09 consistió en el despliegue de aplicaciones backend tanto en PHP como en JSP, permitiendo experimentar con servidores locales (como XAMPP para PHP y Tomcat para JSP) y visualizar el flujo completo de una aplicación web dinámica.

¿Qué aprendí?

Aprendí cómo funciona una aplicación web a nivel backend, desde que el usuario envía una solicitud hasta que recibe una respuesta. Comprendí la lógica del renderizado dinámico de contenido y cómo los lenguajes como PHP y JSP permiten construir aplicaciones web robustas y funcionales. También adquirí nociones básicas sobre cómo configurar y ejecutar servidores web locales.

¿Cómo lo aprendí?

A través de clases teóricas, análisis de la arquitectura cliente-servidor y sesiones prácticas que incluyeron la creación y despliegue de aplicaciones usando PHP y JSP. El laboratorio permitió reforzar el aprendizaje al simular entornos de producción y resolver errores reales durante el despliegue.
