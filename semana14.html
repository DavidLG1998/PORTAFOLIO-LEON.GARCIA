Semana 14: Microservicios en Django

Descripción de los temas aprendidos

Durante la décima cuarta semana se profundizó en la arquitectura de microservicios como una evolución del desarrollo monolítico tradicional. Se presentó Django como una herramienta viable para el desarrollo de servicios independientes y desacoplados que interactúan entre sí de forma modular y escalable. Los principales contenidos abordados fueron: Introducción a los microservicios: Se comprendió el paradigma de microservicios como un enfoque arquitectónico donde cada componente de una aplicación es desarrollado, desplegado y escalado de forma independiente. Se analizaron sus beneficios frente a las arquitecturas monolíticas, incluyendo la mejora en la mantenibilidad, escalabilidad y tolerancia a fallos. Diseño y construcción del primer microservicio con Django: Se creó un microservicio básico empleando Django y Django REST Framework, enfocado en un único dominio funcional. Se aplicaron principios de separación de responsabilidades y diseño modular. Comunicación entre microservicios: Se exploraron mecanismos para la interoperabilidad entre servicios, como llamadas HTTP mediante requests, colas de mensajes y eventos asincrónicos. Se discutieron buenas prácticas para asegurar la confiabilidad y consistencia de la comunicación distribuida. Gestión de bases de datos en microservicios: Se abordó el concepto de base de datos por servicio, donde cada microservicio administra su propio esquema y motor de persistencia, garantizando independencia lógica y física. Implementación de microservicios con Docker y Kubernetes: Se introdujo Docker como herramienta de contenerización para empacar microservicios con todas sus dependencias. Se presentaron los conceptos fundamentales de Kubernetes para la orquestación, despliegue y escalado automático de microservicios en clústeres. Protección y monitoreo: Se estudiaron estrategias para asegurar los microservicios, incluyendo autenticación basada en tokens y control de acceso. Se enfatizó el uso de herramientas de observabilidad y monitoreo, como Prometheus y Grafana, para medir métricas, detectar errores y gestionar el rendimiento.

¿Qué aprendí?

Aprendí a diseñar e implementar microservicios desacoplados utilizando Django, así como a contenerizarlos y orquestarlos mediante Docker y Kubernetes. Asimismo, comprendí cómo asegurar su comunicación, gestionar bases de datos distribuidas y aplicar buenas prácticas de seguridad y monitoreo.

¿Cómo lo aprendí?

El aprendizaje fue altamente práctico, desarrollando microservicios individuales e interconectándolos en un entorno controlado. Se realizaron laboratorios con despliegues locales usando Docker y simulaciones de clústeres, permitiendo una comprensión integral de la infraestructura moderna basada en servicios.
