Semana 11: Introducción al Framework Django y Desarrollo Web Basado en MTV

Descripción de los temas aprendidos

Durante esta semana se abordó la introducción al framework Django, una poderosa herramienta de desarrollo web en Python que permite construir aplicaciones seguras, escalables y mantenibles mediante el patrón arquitectónico MTV (Model-Template-View), una variante del clásico patrón MVC (Model-View-Controller). Los principales contenidos desarrollados fueron: Fundamentos de Django: Se revisó la arquitectura general del framework y su correspondencia con el patrón MVC, en el que: El Modelo representa la estructura de datos. La Vista (View) gestiona la lógica del negocio. Las Plantillas (Template) manejan la presentación. Configuración del entorno: Instalación del entorno virtual y Django mediante pip. Creación de un proyecto inicial con django-admin startproject. Configuración de la integración con servidores web como Apache para su posterior despliegue. Gestión de proyectos y estructuras: Uso de la terminal para crear aplicaciones con python manage.py startapp. Organización de carpetas y archivos esenciales dentro de un proyecto Django. Gestión de URLs, vistas y plantillas: Creación y configuración de URLs mediante el archivo urls.py. Desarrollo de vistas basadas en funciones (Function-Based Views) y vistas basadas en clases (Class-Based Views). Utilización del sistema de plantillas con herencia ({% extends %}), bloques ({% block %}), etiquetas ({% if %}, {% for %}) y filtros (date, length, entre otros). Modelo de datos y consulta a base de datos: Definición de modelos mediante clases que heredan de models.Model. Configuración de campos con distintos tipos (CharField, DateField, IntegerField, etc.). Uso de la API de Django ORM para realizar operaciones CRUD sobre la base de datos (consultas, filtrado, inserción y eliminación).

¿Qué aprendí?

Aprendí a estructurar y configurar proyectos web usando Django, comprendiendo su arquitectura MTV, el ciclo de vida de una petición HTTP en este framework, y la separación de responsabilidades entre modelos, vistas y plantillas. Además, adquirí competencias prácticas para diseñar estructuras de datos y manipularlas eficientemente mediante el ORM de Django.

¿Cómo lo aprendí?

La metodología se basó en la creación práctica de un proyecto web funcional, iniciando desde la línea de comandos, pasando por la definición de rutas, creación de vistas, diseño de interfaces con plantillas, y finalizando con la implementación de modelos y consultas a la base de datos. Este enfoque permitió consolidar de forma progresiva tanto la teoría como la práctica, bajo un entorno de desarrollo profesional.
